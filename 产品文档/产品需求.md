

# 产品需求文档

1. 项目背景：随着团队规模扩大，原有Excel管理问题的方式已无法满足需求。

2. 现有痛点：
   1. 问题数量增多，开发人员处理时经常遗漏
   2. 问题反复出现时，修复轨迹容易丢失
   3. 成员离职后，遗留问题无人修复

3. 团队架构：
   1. 项目负责人：1人
   2. 开发人员：12人
   3. 测试人员：1人
   4. 现场人员：若干

4. 问题处理流程：
   1. 验证环节（4个环境）：开发环境验证 → 测试环境验证 → Beta环境验证 → 线上环境验证
   2. 回退机制：任一验证环节出现问题，推回给开发，标记为"未通过"，记录不通过的环境、描述（包括备注和附件）

5. 需求目标：
   1. 建立问题全生命周期管理机制
   2. 保留问题处理的完整轨迹
   3. 支持人员交接，确保问题不遗漏
   4. 每个项目会有多个需求
   5. 每个项目会有1到多名负责人，负责人可以添加项目成员
   6. 成员角色有开发、测试、部署等，可自定义配置
   7. 每个问题的验证需要展示历史轨迹
   8. 用户可能会有多个任务，但是它需要选择部分任务作为自己的待办，后续修复根据待办进行处理，这样可以提升开发效率。
   9. 由于测试人员的测试两非常大，所以需要做一个“快速测试”的功能，可以快速填写测试结果，并快速切换上下一个，便于查看，测试人员的测试项是开发人员提交的测试项，且当前的测试项必须是同一环境的
   10. 快速测试功能需要在测试通过后立即跳转到下一个(V1.0)
   11. 项目经理可以记录需求并安排开发，同时问题的BUG也可以关联到原始需求，并且需求后续需求有修改需要记录修改轨迹(V1.2)
   12. 每个需求可增加会议纪要，并标识是否讨论，会议纪要可以是附件，也可以是腾讯会议链接(V1.2)
   13. 基本功能包括：用户管理，字典管理（V1.2）
   14. BUG或者需求开发完成后为已完成，但是需要自己体测
