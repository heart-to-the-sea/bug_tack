# 人员管理模块接口文档

## 1. 获取用户列表

**接口路径**: `/api/tack/users`

**请求方法**: `GET`

### 入参

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| page | int | 否 | 页码，默认1 |
| page_size | int | 否 | 每页数量，默认10 |
| keyword | string | 否 | 关键词搜索 |
| role | string | 否 | 角色筛选 |
| project_id | int | 否 | 项目ID筛选 |

### 出参

| 参数名 | 类型 | 说明 |
|--------|------|------|
| code | int | 状态码，0表示成功 |
| message | string | 返回消息 |
| data | object | 用户列表信息 |
| data.list | array | 用户列表 |
| data.list[].id | int | 用户ID |
| data.list[].username | string | 用户名 |
| data.list[].name | string | 姓名 |
| data.list[].email | string | 邮箱 |
| data.list[].phone | string | 电话 |
| data.list[].role | string | 角色 |
| data.list[].avatar | string | 头像URL |
| data.list[].status | string | 状态 |
| data.list[].project_count | int | 参与项目数量 |
| data.list[].issue_count | int | 负责问题数量 |
| data.total | int | 总数量 |

## 8. 人员交接

**接口路径**: `/api/tack/users/{from_id}/transfer-issues`

**请求方法**: `POST`

### 入参

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| from_id | int | 是 | 原负责人ID |
| to_id | int | 是 | 新负责人ID |
| issue_ids | array | 是 | 要转移的问题ID列表 |
| reason | string | 否 | 交接原因 |

### 出参

| 参数名 | 类型 | 说明 |
|--------|------|------|
| code | int | 状态码，0表示成功 |
| message | string | 返回消息 |
| data | object | 交接结果 |
| data.transferred_count | int | 成功转移数量 |
| data.record_id | int | 交接记录ID |
