# 附件管理模块接口文档

## 1. 上传附件

**接口路径**: `/api/tack/attachments/upload`

**请求方法**: `POST`

**Content-Type**: `multipart/form-data`

### 入参

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| file | File | 是 | 上传的文件 |
| issue_id | int | 否 | 关联问题ID |
| type | string | 否 | 附件类型 |

### 出参

| 参数名 | 类型 | 说明 |
|--------|------|------|
| code | int | 状态码，0表示成功 |
| message | string | 返回消息 |
| data | object | 附件信息 |
| data.id | int | 附件ID |
| data.filename | string | 文件名 |
| data.original_name | string | 原始文件名 |
| data.size | int | 文件大小（字节） |
| data.type | string | 文件类型 |
| data.mime_type | string | MIME类型 |
| data.url | string | 文件访问URL |
| data.uploader_id | int | 上传人ID |
| data.created_at | string | 上传时间 |

---

## 2. 批量上传附件

**Content-Type**: `multipart/form-data`

### 入参

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| files | array | 是 | 上传的文件列表 |

### 出参

| 参数名 | 类型 | 说明 |
|--------|------|------|
| code | int | 状态码，0表示成功 |
| message | string | 返回消息 |

| data[].original_name | string | 原始文件名 |
| data[].size | int | 文件大小 |

## 3. 获取附件列表

**接口路径**: `/api/tack/attachments`

**请求方法**: `GET`

### 入参

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| issue_id | int | 否 | 问题ID筛选 |
| type | string | 否 | 附件类型筛选 |
| page | int | 否 | 页码，默认1 |
| page_size | int | 否 | 每页数量，默认10 |

### 出参

| 参数名 | 类型 | 说明 |
|--------|------|------|
| code | int | 状态码，0表示成功 |
| message | string | 返回消息 |
| data | object | 附件列表信息 |
| data.list | array | 附件列表 |
| data.total | int | 总数量 |

## 10. 配置上传规则

**接口路径**: `/api/tack/attachments/config`

**请求方法**: `GET`

### 入参

无

### 出参

| 参数名 | 类型 | 说明 |
|--------|------|------|
| code | int | 状态码，0表示成功 |
| message | string | 返回消息 |
| data | object | 上传配置 |
| data.max_size | int | 最大文件大小（字节） |
| data.allowed_types | array | 允许的文件类型 |
| data.allowed_extensions | array | 允许的文件扩展名 |
| data.max_batch_count | int | 最大批量上传数量 |
